# 写代码 + 自动审阅

在完成代码编写后，自动调用 code-reviewer 审阅代码并根据意见修改，直到代码质量达标。

## 触发条件

当用户的请求包含以下意图时使用此技能：
- "写代码并审阅"、"边写边审"
- "code with review"、"write and review"
- "写完帮我检查"、"写完审阅一下"
- 用户在编码任务中要求质量保证

## 执行流程

### 步骤 1：正常编写代码

按照用户的要求编写代码，与正常的编码流程相同。

### 步骤 2：编写完成后自审阅

代码编写完成后，**不要立即返回给用户**，而是：

1. 调用 `mcp__code-reviewer__review_code` 审阅刚写的代码
2. 检查审阅结果中的中高严重度问题
3. 如果有问题，自动修复
4. 再次审阅确认
5. 重复直到无中高严重度问题或分数 >= 8

### 步骤 3：展示结果

```
✅ 代码编写完成

📊 自审阅结果：X.X/10
🔄 修复了 X 个问题：
- [severity] 问题描述 → 已修复

最终代码已就绪。
```

## 自审阅 Prompt 模板

在步骤 2 中，使用以下 prompt 调用 code-reviewer：

```
调用 mcp__code-reviewer__review_code 工具审阅以下代码。
工具返回的 JSON 中有 result 字段，内含 overall_score、issues、summary。
请将 result 字段的内容原样输出，不要加 markdown 标记：

文件：{file_path}
```{language}
{code}
```
```

## 自动修复 Prompt 模板

如果审阅发现问题，使用以下 prompt 进行修复：

```
你是代码修复专家。根据审阅意见修改代码，只返回修改后的完整代码。

当前代码：
```{language}
{code}
```

审阅问题（总分 {score}/10）：
{issues}

要求：
1. 修复所有中高严重度问题
2. 保持代码功能不变
3. 不要引入新问题
4. 只返回代码块，不要解释
```

## 与 review-current 的配合

`code-with-review` 和 `review-current` 可以配合使用：

1. **编写阶段**：使用 `code-with-review`，Claude 自动审阅
2. **修改阶段**：使用 `review-current`，手动触发审阅
3. **收尾阶段**：使用 `auto-review-loop`，深度优化

## 注意事项

1. **不要过度审阅**：只在代码编写完成后审阅一次，不要每写一行就审阅
2. **保持效率**：如果代码很简单（< 20 行），可以跳过自审阅
3. **用户优先**：如果用户赶时间或明确说不需要审阅，跳过此技能
4. **只修中高**：低严重度问题可以提及但不自动修复，除非用户要求
